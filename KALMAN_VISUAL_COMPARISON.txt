┌─ INPUTS ──────────────────────────────────────────────────────────────────┐
│                                                                            │
│  Each camera frame provides TWO measurements:                             │
│                                                                            │
│  1. center_offset: where target is in frame                              │
│     Range: -1.0 (far left) to +1.0 (far right)                          │
│     Example: 0.15 means target is 15% right of center                    │
│                                                                            │
│     ┌────────────────────────────────────────┐                          │
│     │ ◇ ◇ ◇ ◇  TARGET  ◇ ◇ ◇ ◇ ◇ ◇ ◇ ◇ ◇  │                          │
│     │ (-1.0) (-0.5) (0.0) (0.15) (0.5) (1.0)│                          │
│     │        LEFT     CENTER    RIGHT        │                          │
│     └────────────────────────────────────────┘                          │
│                                                                            │
│  2. area: size of detected target                                        │
│     Range: 50 to 500+ pixels²                                            │
│     Example: 110 pixels² = boat is this big                             │
│                                                                            │
│     ┌────────────────────────────────────────┐                          │
│     │     [Small]    [Medium]    [Large]     │                          │
│     │    (50-100)  (100-200)    (200-500)    │                          │
│     │     FAR        MID         CLOSE        │                          │
│     └────────────────────────────────────────┘                          │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘


┌─ PROCESS: KALMAN FILTER ──────────────────────────────────────────────────┐
│                                                                            │
│  Maintains 4-state vector:                                               │
│  ┌──────────────────────────────────────┐                               │
│  │ state = [offset,                    │                               │
│  │          offset_velocity,            │                               │
│  │          area,                       │                               │
│  │          area_velocity]              │                               │
│  └──────────────────────────────────────┘                               │
│                                                                            │
│  Each frame does TWO steps:                                             │
│                                                                            │
│  STEP 1: PREDICT (time update)                                          │
│  ────────────────────────────────                                        │
│  Uses internal velocity to predict next state                           │
│                                                                            │
│     Position new = Position old + Velocity × dt                         │
│     Area new = Area old + Area_velocity × dt                            │
│                                                                            │
│     Example: dt = 0.1 seconds                                           │
│       offset_prior = 0.10 + 0.5 × 0.1 = 0.15                           │
│       area_prior = 100 + 2 × 0.1 = 100.2                               │
│                                                                            │
│  STEP 2: UPDATE (measurement fusion)                                    │
│  ──────────────────────────────────────                                 │
│  Fuses measurement with prediction using Kalman gain                    │
│                                                                            │
│     innovation = measurement - prediction                               │
│                = [0.20, 105] - [0.15, 100.2]                            │
│                = [0.05, 4.8]  (difference)                              │
│                                                                            │
│     Kalman gain K = how much to trust this measurement                  │
│     (computed from covariance and noise)                                │
│                                                                            │
│     state_new = state_prior + K × innovation                            │
│                = [0.15, v] + K × [0.05, 4.8]                            │
│                                                                            │
│     If K = 0.5: state = [0.175, v', 102.4, ...]                         │
│     (halfway between prediction and measurement)                        │
│                                                                            │
│     Also updates velocity estimate!                                     │
│     If measurement surprised us, velocity changes fast                  │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘


┌─ OUTPUTS ─────────────────────────────────────────────────────────────────┐
│                                                                            │
│  After predict + update:                                                 │
│                                                                            │
│  predicted_offset = state[0]       (filtered position)                  │
│  predicted_area = state[2]         (filtered size)                      │
│  prediction_confidence = 0.0 to 1.0 (how confident)                     │
│                                                                            │
│  These are sent to control loop (same keys as before!)                  │
│  Control doesn't know it's using Kalman - seamless upgrade              │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘


╔════════════════════════════════════════════════════════════════════════════╗
║                           TURN SCENARIO EXAMPLE                            ║
║              Leader Makes Sharp Left Turn: offset goes 0.0 → -0.5         ║
╚════════════════════════════════════════════════════════════════════════════╝


┌─ WITHOUT KALMAN (Old Method) ─────────────────────────────────────────────┐
│                                                                            │
│ Frame 1: measurement = 0.0                                               │
│   velocity = 0.0 / dt = 0.0                                              │
│   prediction = 0.0 + 0.0 × time = 0.0                                    │
│   ✓ Correct                                                              │
│                                                                            │
│ Frame 2 (SHARP TURN): measurement = -0.3                                 │
│   velocity = (-0.3 - 0.0) / 0.1 = -3.0  ← HUGE JUMP                     │
│   prediction = -0.3 + (-3.0) × 0.1 = -0.6  ← OVERSHOOTS                 │
│   ✗ Overreacts, jittery                                                  │
│                                                                            │
│ Frame 3: measurement = -0.5                                               │
│   velocity = (-0.5 - (-0.3)) / 0.1 = -2.0  ← Still unstable             │
│   prediction = -0.5 + (-2.0) × 0.1 = -0.7  ← Still overshooting         │
│   ✗ Still lagging, oscillating                                           │
│                                                                            │
│ Frame 4: measurement = -0.65                                              │
│   velocity = (-0.65 - (-0.5)) / 0.1 = -1.5                               │
│   prediction = -0.65 + (-1.5) × 0.1 = -0.8                               │
│   ✗ Finally catching up (~300ms delay!)                                  │
│                                                                            │
│ RESULT: Oscillations, overshoot, then lag                               │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘


┌─ WITH KALMAN FILTER (New Method) ─────────────────────────────────────────┐
│                                                                            │
│ Frame 1: measurement = 0.0                                               │
│   Update: state = [0.0, 0.0, area, 0.0]                                 │
│   Predict: (no time passed yet)                                          │
│   prediction = 0.0                                                       │
│   ✓ Correct                                                              │
│                                                                            │
│ Frame 2 (SHARP TURN): measurement = -0.3                                 │
│   Predict: state = old_state + velocity × dt = [0.0, 0.0]              │
│   (prediction based on last velocity=0.0)                                │
│                                                                            │
│   Update: innovation = -0.3 - 0.0 = -0.3  (BIG SURPRISE!)               │
│   Kalman gain K becomes HIGH (trust this unexpected measurement)         │
│   state = [0.0, 0.0] + K×[-0.3, ...]                                    │
│         ≈ [-0.15, -1.5, ...]  ← velocity JUMPS to -1.5 ✓              │
│   prediction = -0.15  (good! not -0.3)                                  │
│   ✓ Detects turn, doesn't overshoot                                     │
│                                                                            │
│ Frame 3: measurement = -0.5                                               │
│   Predict: state[0] = -0.15 + (-1.5) × 0.1 = -0.3  ← SMOOTH! ✓         │
│   (uses velocity estimate from Frame 2)                                  │
│                                                                            │
│   Update: innovation = -0.5 - (-0.3) = -0.2  (smaller surprise)         │
│   Kalman gain K becomes LOWER (innovation is expected)                   │
│   state ≈ [-0.4, -1.2, ...]  (settles velocity)                         │
│   prediction = -0.4  (tracking well!)                                    │
│   ✓ Smooth, accurate tracking                                            │
│                                                                            │
│ Frame 4: measurement = -0.65                                              │
│   Predict: state[0] = -0.4 + (-1.2) × 0.1 = -0.52  ← SMOOTH ✓          │
│   Update: innovation = -0.65 - (-0.52) = -0.13  (small)                 │
│   state ≈ [-0.6, -1.0, ...]                                              │
│   prediction = -0.6  (excellent tracking!)                              │
│   ✓ On target, no lag                                                    │
│                                                                            │
│ RESULT: Smooth, fast adaptation, no overshoot or lag                    │
│ (100ms response time vs 300ms without)                                   │
│                                                                            │
└───────────────────────────────────────────────────────────────────────────┘


╔════════════════════════════════════════════════════════════════════════════╗
║                           KEY DIFFERENCE                                   ║
╚════════════════════════════════════════════════════════════════════════════╝

OLD: velocity = (current - previous) / dt
     ├─ Direct calculation
     ├─ Each measurement gives isolated velocity
     └─ Result: Jittery, reactive

NEW: velocity = internal state, continuously filtered
     ├─ Updated by Kalman gain based on innovation
     ├─ Evolves smoothly frame-to-frame
     └─ Result: Smooth, adaptive, predictive

The Kalman "magic":
┌──────────────────────────────────────────────────────┐
│ If innovation (surprise) is LARGE:                   │
│   → Kalman gain K becomes HIGH                       │
│   → Trust the measurement more                       │
│   → Velocity changes quickly                         │
│   → Responds fast to turns ✓                         │
│                                                      │
│ If innovation is SMALL:                              │
│   → Kalman gain K becomes LOW                        │
│   → Trust prediction more                            │
│   → Velocity changes slowly                          │
│   → Smooth trajectory ✓                              │
└──────────────────────────────────────────────────────┘
